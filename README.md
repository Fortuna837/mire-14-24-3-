# Анализ алгоритмов сортировки и поиска
---
## Алгоритмы сортировки
---
### Сортировка выбором (Selection Sort)

**Определение:**  
Алгоритм, который последовательно строит отсортированную часть массива, начиная с начала. На каждом шаге в неотсортированном сегменте отыскивается наименьший элемент, который затем перемещается в конец упорядоченной области.

**Анализ:**  
Алгоритм использует два вложенных цикла. Внешний цикл перебирает элементы, чтобы определить позицию для следующего минимального значения. Внутренний цикл проходит по оставшейся неотсортированной части массива, чтобы найти этот минимальный элемент.

**Временная сложность:**  
O(n²)

**Объяснение временной сложности:**  
Два вложенных цикла выполняют примерно n*(n-1)/2 сравнений. Количество операций растет пропорционально квадрату размера массива, так как при увеличении n в k раз общее время работы увеличивается примерно в k² раз.
---
### Сортировка пузырьком (Bubble Sort)

**Определение:**  
Простой алгоритм, который многократно проходит по списку, сравнивая соседние элементы и меняя их местами, если они находятся в неправильном порядке.

**Анализ:**  
Алгоритм использует вложенные циклы для проходов по массиву. Внешний цикл контролирует количество проходов, а внутренний попарно сравнивает соседние элементы.

**Временная сложность:**  
O(n²)

**Объяснение временной сложности:**  
В худшем случае требуется n-1 проходов, при этом на i-том проходе внутренний цикл выполнит (n-i) сравнений. Суммарное количество операций составляет n(n-1)/2, что дает квадратичную зависимость.
---
### Сортировка вставками (Insertion Sort)

**Определение:**  
Алгоритм, который последовательно берет каждый очередной элемент и вставляет его на нужную позицию в уже отсортированной части массива, сдвигая остальные элементы вправо.

**Анализ:**  
Внешний цикл перебирает элементы, начиная со второго. Для каждого такого элемента внутренний цикл сравнивает его с элементами в отсортированной части и сдвигает их, пока не будет найдена правильная позиция для вставки.

**Временная сложность:**  
O(n²)

**Объяснение временной сложности:**  
В худшем случае для вставки каждого нового элемента приходится сдвигать всю уже отсортированную часть. Количество операций составляет 1 + 2 + ... + (n-1) = n(n-1)/2, что соответствует O(n²).
---
### Сортировка слиянием (Merge Sort)

**Определение:**  
Алгоритм, основанный на парадигме «разделяй и властвуй». Он рекурсивно разбивает исходный массив на две половины, а затем сливает эти подмассивы в упорядоченном виде.

**Анализ:**  
Процесс состоит из двух основных этапов: разделения и слияния. Этап разделения рекурсивно дробит массив. Этап слияния объединяет два отсортированных подмассива в один, последовательно выбирая меньший из первых элементов.

**Временная сложность:**  
O(n log n)

**Объяснение временной сложности:**  
Массив делится пополам на log n уровней. На каждом уровне рекурсии для слияния всех элементов требуется выполнить O(n) операций. Таким образом, общая сложность составляет O(n) * O(log n) = O(n log n).
---
### Сортировка Шелла (Shell Sort)

**Определение:**  
Усовершенствованный вариант сортировки вставками, который сортирует элементы, находящиеся на определенном расстоянии друг от друга. Шаг постепенно уменьшается до единицы.

**Анализ:**  
Алгоритм начинает работу с большого шага (gap), что позволяет элементам быстрее перемещаться в свои позиции. С каждым проходом шаг уменьшается. Эффективность метода зависит от выбранной последовательности шагов.

**Временная сложность:**  
O(n^(3/2)) для последовательности Кнута

**Объяснение временной сложности:**  
При использовании оптимальных последовательностей шагов (как у Кнута) количество операций растет пропорционально n в степени 1.5. Это лучше квадратичной сложности, но хуже чем O(n log n).
---
### Быстрая сортировка (Quick Sort)

**Определение:**  
Алгоритм типа «разделяй и властвуй», который выбирает опорный элемент и перераспределяет другие элементы относительно него: меньшие размещаются слева, большие — справа.

**Анализ:**  
Ключевой этап — разделение массива относительно опорного элемента. Затем процесс рекурсивно применяется к двум полученным подмассивам. Эффективность зависит от выбора опорного элемента.

**Временная сложность:**  
O(n log n)

**Объяснение временной сложности:**  
При сбалансированном разбиении массива глубина рекурсии составляет O(log n), а на каждом уровне выполняется O(n) работы. Таким образом, общая сложность равна O(n) * O(log n) = O(n log n).
---
### Пирамидальная сортировка (Heap Sort)

**Определение:**  
Алгоритм, который использует структуру данных «двоичная куча». Он сначала преобразует массив в max-кучу, где наибольший элемент находится в корне, затем последовательно извлекает его и перестраивает кучу.

**Анализ:**  
Процесс состоит из двух этапов: построения кучи и последовательного извлечения максимальных элементов. Каждое извлечение требует перестройки кучи для поддержания ее свойств.

**Временная сложность:**  
O(n log n)

**Объяснение временной сложности:**  
Построение кучи выполняется за O(n) операций. Затем выполняется n извлечений максимального элемента, каждое из которых требует O(log n) операций для перестройки кучи. Итоговая сложность: O(n) + O(n log n) = O(n log n).
---
## Алгоритмы поиска
---
### Линейный поиск (Linear Search)

**Определение:**  
Простой алгоритм, который последовательно проверяет каждый элемент массива до тех пор, пока не найдет искомый элемент или не достигнет конца массива.

**Анализ:**  
Алгоритм начинает с первого элемента и последовательно движется к концу массива, сравнивая текущий элемент с искомым значением.

**Временная сложность:**  
O(n)

**Объяснение временной сложности:**  
В худшем случае требуется проверить все n элементов массива. При увеличении размера массива в k раз время поиска в худшем случае увеличивается в k раз.
---
### Бинарный поиск (Binary Search)

**Определение:**  
Алгоритм для поиска элемента в отсортированном массиве, который на каждом шаге делит область поиска пополам.

**Анализ:**  
Алгоритм сравнивает искомый элемент со средним элементом текущей области поиска. В зависимости от результата сравнения поиск продолжается в левой или правой половине массива.

**Временная сложность:**  
O(log n)

**Объяснение временной сложности:**  
На каждом шаге область поиска уменьшается вдвое. Количество шагов, необходимых для сокращения n элементов до 1, равно log₂n, что дает логарифмическую сложность.
---
### Интерполяционный поиск (Interpolation Search)

**Определение:**  
Алгоритм поиска в отсортированном массиве, который использует значения элементов для предсказания позиции искомого элемента.

**Анализ:**  
Вместо деления области поиска пополам, алгоритм вычисляет вероятную позицию элемента на основе его значения и значений на границах текущей области поиска.

**Временная сложность:**  
O(log log n) при равномерном распределении данных

**Объяснение временной сложности:**  
При равномерном распределении данных на каждом шаге область поиска сокращается значительно быстрее, чем в бинарном поиске. Количество шагов пропорционально log log n.
---
### Поиск Фибоначчи (Fibonacci Search)

**Определение:**  
Алгоритм поиска в отсортированном массиве, который использует числа Фибоначчи для определения точек разделения массива.

**Анализ:**  
Алгоритм использует числа Фибоначчи для определения позиций сравнения. На каждом шаге область поиска сокращается с использованием свойств последовательности Фибоначчи.

**Временная сложность:**  
O(log n)

**Объяснение временной сложности:**  
Количество итераций цикла пропорционально номеру числа Фибоначчи, которое больше или равно n. Так как числа Фибоначчи растут экспоненциально, количество итераций составляет O(log n).
